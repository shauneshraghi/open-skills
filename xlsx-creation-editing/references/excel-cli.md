# Excel Command-Line Switches (excel.exe)

Command-line switches control how Excel starts and what it does on launch.

**General syntax:**

```
excel.exe [switches] [filepath]
```

Unlike Word's `/m<macroname>`, Excel has **no CLI switch to run a named macro** on
startup. Macro execution requires COM automation, a workbook with an embedded
`Auto_Open` or `Workbook_Open` macro, or the `/a` add-in loader.

---

## Switches Reference

| Switch | Description |
|---|---|
| `workbook path \| file name` | No switch needed. Starts Excel and opens the specified file. Supports local paths and HTTP/SharePoint URLs. |
| `/r workbook` | Opens the specified workbook as **read-only**. |
| `/t workbook` | Starts Excel and opens the specified file as a **template** (new workbook based on the file). |
| `/n workbook` | Identical to `/t` — opens the specified file as a template. |
| `/e` or `/embed` | Prevents the Excel startup screen from appearing and suppresses the new blank workbook. Use for embedded/OLE automation scenarios. |
| `/p "folder path"` | Sets the specified folder as the **active working folder** (the folder shown in Save As). |
| `/s` or `/safemode` | Forces Excel to bypass all files stored in startup directories (e.g., the default XLStart folder). Useful for troubleshooting add-in conflicts. |
| `/m` | Creates a new workbook containing a single **XLM macro sheet** (legacy Excel 4.0 macro). |
| `/a progID` | Starts Excel and loads the **Automation add-in** identified by the given COM ProgID. |
| `/x` | Starts a **new instance** (separate process) of Excel, even if one is already running. |

## Invocation examples

```bat
REM Open a workbook
excel.exe "C:\Reports\quarterly.xlsx"

REM Open a workbook from SharePoint
excel.exe http://MySite/Documents/Book1.xlsx

REM Open read-only
excel.exe /r "C:\data\reference.xlsx"

REM Open as a template (new workbook based on the file)
excel.exe /t "C:\templates\budget_template.xlsx"

REM Suppress startup screen (OLE/embedding scenarios)
excel.exe /e

REM Set working folder
excel.exe /p "C:\My Reports"

REM Safe mode (skip XLStart add-ins)
excel.exe /s

REM Load an Automation add-in
excel.exe /a MyAddIn.Class.1

REM Start a new isolated Excel instance
excel.exe /x "C:\data\isolated.xlsx"
```

## Running VBA macros from the command line

Excel.exe has no equivalent to Word's `/m<macroname>`. To run a macro on startup:

### Option A — Embed `Auto_Open` in the workbook (no COM needed)

Excel calls `Auto_Open` automatically when a workbook opens. Combine with the
workbook path argument:

```bat
excel.exe "C:\automation\my_report.xlsm"
```

The workbook's `Auto_Open` Sub runs immediately after the file opens.

### Option B — VBScript COM bridge (generated by `scripts/vba_runner.py`)

```vbs
' Generated by vba_runner.py generate-vbs
Dim oExcel, oWb, oModule
Set oExcel = CreateObject("Excel.Application")
oExcel.Visible = False
Set oWb = oExcel.Workbooks.Open("C:\data\report.xlsx")
Set oModule = oWb.VBProject.VBComponents.Add(1)
oModule.CodeModule.AddFromString "<vba_code>"
oExcel.Run "SubName"
oWb.Save
oWb.Close False
oExcel.Quit
```

Run with:

```bat
cscript //Nologo run_macro.vbs
```

> **Note:** Programmatic access to the VBA project model must be enabled in Excel's
> Trust Center (File → Options → Trust Center → Macro Settings →
> "Trust access to the VBA project object model").

### Option C — pywin32 COM (Windows + pywin32 required)

```python
import win32com.client
excel = win32com.client.Dispatch("Excel.Application")
wb = excel.Workbooks.Open(r"C:\data\report.xlsx")
excel.Run("MyMacro")
wb.Save()
wb.Close(False)
excel.Quit()
```

## Invoking via `scripts/vba_runner.py`

```bash
# Check excel.exe is on PATH
python scripts/vba_runner.py check

# Generate a VBScript runner
python scripts/vba_runner.py generate-vbs \
    --wb report.xlsx --code my_macro.vba --sub RunReport --out run_macro.vbs

# Invoke excel.exe with a workbook (Auto_Open macro in the file)
python scripts/vba_runner.py invoke --wb report.xlsm

# Open read-only
python scripts/vba_runner.py invoke --wb data.xlsx --read-only

# Safe mode (skip XLStart)
python scripts/vba_runner.py invoke --wb data.xlsx --safe

# New isolated instance
python scripts/vba_runner.py invoke --wb data.xlsx --new-instance

# Run via COM automation (Windows + pywin32 required)
python scripts/vba_runner.py run-com --wb report.xlsx --code my_macro.vba --sub RunReport
```

## Compatibility notes

- All switches are **case-insensitive** on Windows.
- Paths with spaces must be **quoted**.
- `/s` and `/safemode` are equivalent; either form works.
- `/e` and `/embed` are equivalent; either form works.
- On Linux/macOS, `excel.exe` requires Wine. COM automation scenarios work best on a
  Windows host.
- For VBA project model access, Excel's Trust Center must have "Trust access to the
  VBA project object model" enabled.
- `Auto_Open` fires for all open methods; `Workbook_Open` (in `ThisWorkbook`) fires
  for all open methods including programmatic opens via COM.
