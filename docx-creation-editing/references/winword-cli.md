# Word Command-Line Switches (winword.exe)

Command-line switches control how Word starts and what it does on launch.

**General syntax:**

```
winword.exe [switches] [filepath]
```

Switches that take a value written with no space between the switch and the value:
`/mmacroname`, `/ttemplatename`, `/ladd-in`. Switches that take a separate filename
argument use a space: `/t filename`, `/f filename`.

---

## Switches Reference

| Switch | Description |
|---|---|
| `/safe` | Start Word in Safe Mode (disables add-ins, customizations, and AutoExec macros). |
| `/q` | Start Word without displaying the Word splash screen. |
| `/n` | Start a new instance of Word with no document open. Documents opened in each instance will not appear in the Switch Windows list of other instances. |
| `/w` | Start a new instance of Word with a blank document. Documents opened in each instance will not appear in the Switch Windows list of other instances. |
| `/ttemplatename` | Start Word with a new document based on a template other than Normal. No space between `/t` and the template name. Example: `/tc:\Myfax.dotx` |
| `/t filename` | Start Word and open an existing file (space between `/t` and filename). Can open multiple files: `/t c:\A.docx c:\B.docx` |
| `/f filename` | Start Word with a new document based on an existing file. |
| `/h http://filename` | Start Word and open a read-only copy of a document stored on a SharePoint site. Has no effect if the document is already checked out to you. |
| `/m` | Start Word without running any AutoExec macros. |
| `/mmacroname` | Start Word and run a specific macro. Also prevents AutoExec macros from running. No space between `/m` and the macro name. Example: `/mSalelead` |
| `/a` | Start Word and prevent add-ins and global templates (including Normal.dotm) from loading automatically. Also locks the setting files. |
| `/ladd-in` | Start Word and load a specific Word add-in. No space between `/l` and the add-in name. Example: `/lc:\Sales.dll` |
| `/r` | Re-register Word in the Windows registry, update file associations, then close. Do not combine with document paths. |
| `/pxslt` | Start Word and open an existing XML document based on the specified XSLT. Example: `/pc:\MyTransform.xsl c:\Data.xml` |
| `/x` | Start Word from the OS shell so Word responds to only one DDE request (e.g. to print a document programmatically). |
| `/ztemplatename` | Behaves like `/ttemplatename` but generates both a Startup and a New event. (`/t` generates only a Startup event.) |

---

## Key distinctions

- **`/t` with no space** (`/ttemplatename`) — open a *new* document based on a named template
- **`/t` with space** (`/t filename`) — open an *existing* file
- **`/m` with no name** — suppress AutoExec macros only
- **`/m` with name** (`/mmacroname`) — suppress AutoExec macros *and* run the named macro
- **`/q`** — hides the splash screen; does **not** suppress macros

## Automation patterns

### Run a named macro stored in the document

The macro must already exist in the .docm VBA project. No space between `/m` and the name:

```bat
winword.exe /mRunReport "C:\Reports\quarterly.docm"
```

### Load a .dotm template that provides the macro, then open a document

No space between `/t` and the template path:

```bat
winword.exe /t"C:\templates\automation.dotm"
```

To open a document with the template loaded, open the document separately after the
template's `AutoOpen` macro has set up the environment, or embed the logic in the
template's `AutoOpen` to open the target document programmatically.

### Suppress AutoExec macros (safe open for inspection)

```bat
winword.exe /m "C:\data\untrusted.docx"
```

### Open with all add-ins and templates disabled

```bat
winword.exe /a "C:\data\trusted.docx"
```

### Start in Safe Mode

```bat
winword.exe /safe
```

### Open an existing file in a new instance

```bat
winword.exe /n /t "C:\data\input.docx"
```

## Running VBA from a VBScript wrapper (no macro pre-embedded)

When the macro is not pre-embedded, use `scripts/vba_runner.py generate-vbs` to
create a COM-automation wrapper:

```vbs
' Generated by vba_runner.py generate-vbs
Dim oWord, oDoc, oModule
Set oWord = CreateObject("Word.Application")
oWord.Visible = False
Set oDoc = oWord.Documents.Open("C:\data\input.docx")
Set oModule = oDoc.VBProject.VBComponents.Add(1)
oModule.CodeModule.AddFromString "<vba_code>"
oWord.Run "MySubName"
oDoc.Save
oDoc.Close False
oWord.Quit
```

Run with:

```bat
cscript //Nologo run_macro.vbs
```

> **Note:** Programmatic VBA project access requires enabling "Trust access to the
> VBA project object model" in Word's Trust Center (File → Options → Trust Center →
> Macro Settings).

## Invoking via `scripts/vba_runner.py`

```bash
# Check winword.exe is on PATH
python scripts/vba_runner.py check

# Generate a VBScript runner
python scripts/vba_runner.py generate-vbs \
    --doc report.docx --code my_macro.vba --sub RunReport --out run_macro.vbs

# Invoke winword.exe with /m (macro must be in the .docm)
python scripts/vba_runner.py invoke --doc report.docm --macro RunReport

# Disable add-ins and suppress splash
python scripts/vba_runner.py invoke --doc input.docx --no-addins

# Run via COM automation (Windows + pywin32 required)
python scripts/vba_runner.py run-com --doc report.docx --code my_macro.vba --sub RunReport
```

## Compatibility notes

- All switches are **case-insensitive** on Windows.
- Paths with spaces must be **quoted**: `winword.exe /m"My Macro" "C:\My Docs\file.docx"`.
- The `/r` switch is non-interactive and exits immediately; do not combine with document paths.
- On Linux/macOS, `winword.exe` requires Wine. Most automation scenarios work better via
  the VBScript bridge or pywin32 COM on a Windows host.
- Security: because templates and macros can contain viruses, run antivirus software, set
  macro security to High, and clear "Trust all installed add-ins and templates" unless you
  control the source.
