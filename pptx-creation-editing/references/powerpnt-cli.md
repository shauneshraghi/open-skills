# PowerPoint Command-Line Switches (POWERPNT.exe)

Command-line switches control how PowerPoint starts and what it does on launch.

**General syntax:**

```
POWERPNT.exe [switches] [filepath]
```

---

## Switches Reference

| Switch | Description |
|---|---|
| `/B` | Start PowerPoint with a new blank presentation. |
| `/C <file>` | Start PowerPoint with the specified file open and also start Windows NetMeeting conferencing. Omitting the file starts PowerPoint with no presentation and no conferencing. |
| `/EMBEDDING` | Start PowerPoint without displaying the program window. Used by developers when testing add-ins. |
| `/M <file> "<MacroName>"` | Open `<file>` and run the named macro. The presentation file comes immediately after `/M`; the macro name follows as a separate quoted argument. The file must be a macro-enabled format (`.pptm`). |
| `/N [<template.potx>]` | Start PowerPoint and create a new presentation. If a `.potx` template path is supplied, the new presentation is based on that template; otherwise the blank template is used. |
| `/O <file1> [<file2> ...]` | Start PowerPoint and open a comma-separated list of files. |
| `/P <file>` | Print the presentation to the default printer using default settings. PowerPoint shows the Print dialog before printing, then closes after the job completes. |
| `/PT "<PrinterName>" "" "" "<file>"` | Print the presentation to the named printer without showing the PowerPoint window. The two empty quoted arguments are required positional placeholders. |
| `/PWO <file>` | Print the specified file but show the Print dialog box. |
| `/RESTORE` | Restore PowerPoint to the state it was in before an abnormal close, recovering presentations and program state where possible. |
| `/S <file>` | Open the specified presentation directly as a slide show. |

## Invocation examples

```bat
REM New blank presentation
POWERPNT.exe /B

REM Open a presentation as a slide show
POWERPNT.exe /S "C:\Decks\keynote.pptx"

REM New presentation based on a template
POWERPNT.exe /N "C:\Templates\corporate.potx"

REM Open multiple files
POWERPNT.exe /O "C:\Decks\deck1.pptx" "C:\Decks\deck2.pptx"

REM Run a macro in a macro-enabled presentation
POWERPNT.exe /M "C:\Decks\report.pptm" "BuildSlides"

REM Print to default printer (shows dialog)
POWERPNT.exe /P "C:\Decks\report.pptx"

REM Print to specific printer silently
POWERPNT.exe /PT "HP LaserJet" "" "" "C:\Decks\report.pptx"

REM Restore after abnormal close
POWERPNT.exe /RESTORE
```

## Running VBA macros from the command line

PowerPoint supports the `/M` switch to run a named macro, unlike Excel which has no macro-run switch.

### Option A — `/M` switch (macro pre-embedded in a .pptm)

The macro must already exist in the `.pptm` VBA project. File path comes immediately after `/M`, macro name follows as a separate argument:

```bat
POWERPNT.exe /M "C:\Decks\report.pptm" "BuildSlides"
```

The presentation opens, the macro runs, and PowerPoint closes.

### Option B — VBScript COM bridge (generated by `scripts/vba_runner.py`)

When the macro is not pre-embedded, use the VBScript bridge to inject code at runtime:

```vbs
' Generated by vba_runner.py generate-vbs
Dim oPPT, oPrs, oModule
Set oPPT = CreateObject("PowerPoint.Application")
oPPT.Visible = True
Set oPrs = oPPT.Presentations.Open("C:\Decks\report.pptx")
Set oModule = oPrs.VBProject.VBComponents.Add(1)
oModule.CodeModule.AddFromString "<vba_code>"
oPPT.Run "MacroName"
oPrs.Save
oPrs.Close
oPPT.Quit
```

Run with:

```bat
cscript //Nologo run_macro.vbs
```

> **Note:** PowerPoint's COM automation typically requires `Visible = True`. Programmatic
> VBA project access requires enabling "Trust access to the VBA project object model" in
> PowerPoint's Trust Center (File → Options → Trust Center → Macro Settings).

### Option C — pywin32 COM (Windows + pywin32 required)

```python
import win32com.client
ppt = win32com.client.Dispatch("PowerPoint.Application")
ppt.Visible = True
prs = ppt.Presentations.Open(r"C:\Decks\report.pptm")
ppt.Run("BuildSlides")
prs.Save()
prs.Close()
ppt.Quit()
```

## Presentation_Open vs Auto_Open

Unlike Word (`AutoOpen`) and Excel (`Auto_Open`), PowerPoint has no `Auto_Open` Sub.
To run code when a presentation opens, use `Private Sub Presentation_Open()` in the
`ThisPresentation` module of the `.pptm` VBA project.

```vba
' In ThisPresentation module
Private Sub Presentation_Open()
    BuildSlides
End Sub
```

## Invoking via `scripts/vba_runner.py`

```bash
# Check POWERPNT.exe is on PATH
python scripts/vba_runner.py check

# Generate a VBScript runner
python scripts/vba_runner.py generate-vbs \
    --prs report.pptx --code my_macro.vba --sub BuildSlides --out run_macro.vbs

# Invoke POWERPNT.exe with /M (macro must be in the .pptm)
python scripts/vba_runner.py invoke --prs report.pptm --macro BuildSlides

# Open as slide show
python scripts/vba_runner.py invoke --prs keynote.pptx --slide-show

# Run via COM automation (Windows + pywin32 required)
python scripts/vba_runner.py run-com --prs report.pptx --code my_macro.vba --sub BuildSlides
```

## Compatibility notes

- All switches are **case-insensitive** on Windows.
- Paths with spaces must be **quoted**.
- `/M` requires a macro-enabled file (`.pptm`); it will not run macros in `.pptx`.
- PowerPoint COM automation typically requires `Visible = True`; fully headless operation is unreliable.
- On Linux/macOS, `POWERPNT.exe` requires Wine. COM automation is best done on a Windows host.
- For VBA project model access, PowerPoint's Trust Center must have "Trust access to the VBA project object model" enabled.
